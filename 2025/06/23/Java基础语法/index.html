自动转换：

![[Pasted image 20250618210902.png]]
1.byte,short,char之间不会相互转换，他们三者在计算时首先转换为int类型。
2.boolean类型不能与其它数据类型运算。boolean类型数据只允许取值true和false，无null，不可以转换为其它的数据类型
3.当把任何基本数据类型的值和字符串(String)进行连接运算时(+)，基本数据类型的值将自动转化为字符串(String)类型。

常见错误：

报错类型 常见原因 解决方法
NullPointerException 调用了null对象的方法或属性 检查对象是否为null后再操作
ArrayIndexOutOfBoundsException 数组索引越界 检查数组长度和索引范围
ClassCastException 不正确的类型强制转换 使用instanceof进行类型检查
IllegalArgumentException 方法接收到非法参数 检查参数有效性并添加验证
IOException 文件/流操作失败 检查文件路径、权限或资源是否关闭
NumberFormatException 字符串转换为数字格式不正确 使用try-catch或正则校验输入
NoClassDefFoundError 类定义在编译时存在但运行时缺失 检查类路径和依赖配置
StackOverflowError 递归调用过深或无限循环 检查递归终止条件
ConcurrentModificationException 遍历集合时被修改 使用迭代器或并发集合类
OutOfMemoryError 堆内存不足 调整JVM内存参数或优化代码

自定义异常类

当使用 System.out.println(e) 打印一个对象时，会自动调用该对象的 toString() 方法

  • 处理运行时异常时，采用逻辑去合理规避同时辅助try-catch处理
  • 在多重catch块后面，可以加一个catch（Exception）来处理可能会被一楼的异常
  • 对于不确定的代码，也可以加上try-catch，处理潜在的异常
  • 尽量去处理异常，切记只是简单地调用printStack Trace去打印输出
  • 具体如何处理异常，要根据不同的业务需求和异常类型去决定
  • 尽量添加finally语句块去释放占用的资源
